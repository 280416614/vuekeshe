一、 用户与数据管理模块

1.1 本地用户数据管理功能
①系统基于localStorage实现本地数据持久化，无需网络请求及后端数据库。
②系统初始化时自动创建默认用户数据，包含默认用户名及初始设置。
③用户可随时在设置页面修改个人学习参数，包括专注时长等。

1.2 个人学习参数配置功能
①用户可以自定义专注时长，默认为25分钟。


二、 学习计划管理模块

2.1 学习任务的创建与维护功能
①用户可以在“今日计划”页面创建新的学习任务。
②创建任务时，用户需填写任务标题，并可设定预估所需的番茄钟数量。
③系统支持对已创建的任务进行编辑修改和删除操作。
④任务列表按状态进行分组展示，分为“待完成”与“已完成”两组。

2.2 任务的进度追踪功能
①每个任务卡片上以进度条形式直观展示当前完成进度。
②进度计算公式为：已完成番茄数 / 预估番茄数。
③当任务完成番茄数达到预估番茄数时，任务状态自动变为已完成。

2.3 任务的筛选与排序功能
①用户可以通过顶部筛选切换查看全部任务或仅查看今日创建的任务。
②任务列表默认按创建时间倒序排列，最新创建的任务显示在最前面。

三、 专注计时执行模块（核心）

3.1 计时器的启动与状态控制功能
①用户点击任务卡片上的“开始专注”按钮，进入小窗口计时页面。
②计时页面以大字号居中显示当前剩余时间，格式为MM:SS。
③用户可通过底部控制按钮对计时器进行开始、暂停和重置操作。
④计时器运行时，页面标题动态更新剩余时间，便于用户切换浏览器标签页时依然可见。

3.2 番茄钟的完成与自动流转功能
①当倒计时归零时，系统自动记录一条专注日志。
②系统自动将当前任务的已完成番茄数加一。
③系统自动弹出浏览器桌面通知，提示用户“专注完成”。


3.3 计时器的安全与防误操作功能
①用户如果在计时未结束时点击返回按钮，系统弹出确认框，提示“专注尚未完成，确定要退出吗？”。
②计时器组件在页面卸载（关闭或刷新）前自动清除定时器，防止内存泄漏。

四、 数据统计与可视化分析模块


4.2 任务耗时排行榜功能
①系统自动统计每个任务的总专注时长（完成任务的所有番茄钟总耗时）。
②系统按总耗时降序排列所有任务。
③排行榜中每个条目显示任务名称、总耗时（分钟）和完成日期。

五、 成绩报告与反馈模块

5.1 任务完成报告功能
①用户完成一个任务的所有番茄钟后，系统自动生成该任务的简要报告。
②报告内容包括任务总耗时、实际所用番茄钟数、开始和完成时间戳。
③用户可以在任务列表中点击已完成任务查看其完成报告。

5.2 学习提醒与激励功能
①系统在用户连续完成多个番茄钟后，显示鼓励性文案（如“太棒了，你已经专注了XX小时！”）。

技术栈：
前端框架：Vue 3

UI 组件库：Element Plus

路由管理：Vue Router 

状态管理：使用全局 reactive 对象 + watch 即可，降低复杂度

数据持久化：localStorage
